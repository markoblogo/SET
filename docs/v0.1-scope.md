# SET v0.1 Scope

This document defines the first version of the orchestrator.

## Goal

Ship one stable GitHub Action entrypoint that orchestrates repo-facing tooling without reimplementing tool logic.

## v0.1 principle

Keep `SET` thin.

`SET` should install tools and run them in a predictable order.
It should not absorb business logic from `agentsgen`, `git-tweet`, or future tools.

## What v0.1 includes

### Supported tool family

Only `agentsgen` in v0.1.

### Supported operations

- install `agentsgen`
- run `agentsgen init`
- optionally run `agentsgen pack`
- optionally run `agentsgen check --all --ci`

### Initial inputs

Suggested minimal input surface:

```yaml
with:
  agentsgen: "true"
  init: "true"
  pack: "false"
  check: "false"
  autodetect: "true"
  preset: ""
  path: "."
```

Notes:
- `agentsgen` is the feature family switch
- `preset` is optional and only affects init
- `path` defaults to repo root

## What v0.1 explicitly does not include

- no `git-tweet` orchestration
- no `LLMO` migration features yet
- no dashboard backend
- no direct GitHub API repo mutation
- no config registry service
- no multi-tool dependency graph beyond a simple ordered flow

## Execution order

The default execution order should be:

1. install runtime
2. run `agentsgen init` if enabled
3. run `agentsgen pack` if enabled
4. run `agentsgen check --all --ci` if enabled

## Why this scope

This gives one working entrypoint quickly while keeping risk low.

It also creates the stable place where future tools can be added one by one.

## v0.1 success criteria

- one reusable Action in `SET`
- works in a clean target repo with only inputs
- no duplicated `agentsgen` logic in `SET`
- output is readable in GitHub Actions logs
- adding a second tool later does not require redesign

## Next after v0.1

### v0.2 candidates

- registry-driven tool definitions
- presets such as `minimal`, `repo-docs`, `full`
- support for `agentsgen analyze`
- support for `agentsgen meta`

### later

- `git-tweet` integration
- central repo config schema
- read-only Lab dashboard
- PR-based config apply
